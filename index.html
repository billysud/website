SHA-256 hashing and Blockchain

Embedded Software

A two part project where I wrote code that takes in any type of file smaller than 65KB and returns the SHA-256 hash of it. This was accomplished with techniques such as parallel computing and a finite state machine. The SHA-256 is a cryptographically safe algorithm that uses an known initial hash values and compresses the hashes and combines them with contents from the file. The second part of the project involved modifying the SHA-256 hasher to try 16 nonce values simultaneously to check to see if this fits the ending hash of the previous block to verify the blockchain. Due to academic integrity reasons this code is not public.

Huffman Compression and Decompression

Script

A popular algorithm derived in the 50's that uses a similar approach to compressing a file by taking the most used elements of a file and prioritizing those elements for a short length. A Huffman tree was created and used here to take in an arbitrary file of any type and reduces the size of it. Then another Huffman tree would be used to decode the file into the original file. This means that using the UNIX command diff between the original file and the result of the compressed and then decompressed file should result in no output.
